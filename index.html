CSS

The language we all love to hate

* I always complain about CSS and how silly it is * the problem with CSS is not that it's a silly language, the problem with CSS is that it offers too many options * you can write inline (please don't), you can have the style tag and throw there some CSS (please don't), or you can write it in a separate file * even when you're writing it in a different file, the cascading nature of it allows us to mess it up really easy

What's the easiest way to fix a CSS issue?

...just increase specificity or use !important

(please don't!)

* funny thing is that something as simple as CSS can render your website unusable for the users * if we don't want to spend too much time trying to fix CSS we should have some rules in place, some standards so we all speak the same CSS

No. 1 enemy for me?

Specificty*

#I .can't #even { display: none!important; }

* Specificity is the means by which a browser decides which CSS property values are the most relevant to an element and therefore will be applied.
* specificity makes everything brittle * relying on context for a style makes everything impossible to reuse * so you have to add more styles, more exceptions, trying to fig a square peg through a round hole

Introducing

BEM

.block__element--modifier

What is BEM?

  • it's a naming methodology
  • created by Yandex
  • it fakes a more scoped way of developing your CSS
  • it forces the developer to think in components rather than pages
* think about pages being built as using lego * pages are built using the same components (bricks) just arranged in a different way * a group of bricks (a component) can form a larger brick (a new, more complex component)

Inefficient way:

<section class="autobid-tags">
  <h1 class="title">Our business loans can suit almost any business purpose:</h1>

  <ul class="tags">
    <li>Expansion</li>
    ...
  </ul>
  <p>And much more, just tell us in the application.</p>
  <a href="" class="button-blue">Register Now</a>
</section>

.block__element--modifier

<section class="content-tags">
  <h1 class="content-tags__title">Our business loans can suit almost any business purpose:</h1>

  <ul class="content-tags__tags">
    <li>Expansion</li>
    ...
  </ul>
  <p>And much more, just tell us in the application.</p>
  <a href="" class="btn btn--regular btn--primary">Register Now</a>
</section>
Example of using components and modifiers:
<div class="message">
  <h3 class="message__title">Title of the message</h3>
  <p class="message__content">Lorem ipsum [...]</p>
</div>
<div class="message message--success">
  <h3 class="message__title">Title of the message</h3>
  <p class="message__content">Lorem ipsum [...]</p>
</div>
Example of Sass implementation
.message {
  ...
  &__title { ... }
  &__content { ... }
}

// differences between the default and the modifier
.message--success {
  ...
  &__title { ... }
  &__content { ... }

Prefixed classes

Utility classes

  • quick to use classes
  • prefixed with .u-<UTILITY-NAME>
  • can act as quick modifiers
  • they only have one strict result


Examples:

.u-center-text { text-align: center; }
.u-block { display: block; }
.u-relative { position: relative; }
.u-img-round {
  border-radius: 50%;
  display: block;
  width: 100%;
  height: auto;
}

Status classes

  • Used for showing different states of interactive objects
  • prefixed with .is-<STATE>
  • applied/removed from elements using JavaScript


Examples:

.is-active { display: block; }
.is-hidden { display: none; }
.is-transparent { visibility: hidden; }
.is-visible { opacity: 1; }

Binding JavaScript

  • never use classes for both styling and binding JS
  • prefix classes with .js-<FUNCTION>


Examples:

<span class="content-tags__tags js-insert-tags">...</span>

Let's recap

  • don't use inline CSS
  • don't increase specificity if not needed
  • never EVER use !important
  • think about pages as a collection of small components
  • use BEM to create small islands (components) of styles
  • styling of an element should not rely on the context, except if that context is defined by BEM
  • use .is-STATUS to define the status
  • use .js-FUNCTION to bind JS
